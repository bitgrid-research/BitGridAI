# 05.2.3.2 Web-UI

Verantwortung: zeigt Live-State, Timeline, Explain-Sessions und erlaubt Overrides sowie Research-Opt-ins. Arbeitet nur gegen lokale REST/WS-Schnittstellen. Das Cockpit mit klaren Anzeigen.

## Struktur

- **State Dashboard:** Live-Kacheln fuer PV/Netz/Speicher/Miner, Health-Status.
- **Timeline/Preview View:** Zeitachsen mit Forecast/History und What-if-Resultaten.
- **Actions Panel:** Overrides (Boost/Stop/Limit) mit TTL, Command-Feedback.
- **Research/Consent Panel:** Opt-in/Opt-out und Export-Ausloesung, Statusanzeigen.

## Schnittstellen

- **Provided:** UI-Interaktionen (Override/Opt-in/Export), Visualisierung von State/Decision/Explain.
- **Required:** WS/REST vom API-Layer, Explain-Sessions, Preview-Resultate, Health/Command-Feedback.

## Ablauf (vereinfacht)

1) UI verbindet WS -> erhaelt laufende State/Decision/Explain-Events.  
2) Nutzer triggert Override -> UI sendet an API -> zeigt Bestaetigung/Fehler.  
3) Nutzer startet Preview -> UI zeigt Simulationsergebnis.  
4) Nutzer setzt Opt-in -> UI zeigt Export-Status/Hash bei Erfolg.

## Qualitaet und Betrieb

- Offline/local-first; keine Third-Party-Calls.  
- Klare Zustandsanzeige fuer laufende Overrides (command_id, valid_until).  
- Fehler- und Health-Anzeigen fuer Feldadapter und Core.
